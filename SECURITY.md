# Security Policy

## Supported Versions

BugSlyce is an early MVP. Security fixes are currently made against the latest
`main` branch and current tagged release where practical.

## Reporting a Vulnerability

Please do not open a public GitHub issue for sensitive vulnerabilities in
BugSlyce itself.

Instead, report privately by contacting the maintainer. Use the public GitHub
profile/contact route if no dedicated security email is listed.

Do not include real third-party target evidence, credentials, API keys,
cookies, tokens, or private bug bounty programme data unless explicitly
requested.

## Scope

This policy covers vulnerabilities in BugSlyce itself.

It does not cover third-party systems, labs, CTF machines, bug bounty targets,
or findings discovered while using BugSlyce.

## Safety

BugSlyce is intended only for authorised testing. Reports involving
unauthorised targets or requests for exploitation guidance may be closed
without action.
